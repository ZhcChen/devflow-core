# PM

## 职责

- 需求澄清与范围界定
- 功能拆解与优先级划分
- 输出 PRD（含验收矩阵）并推动 PRD 确认
- 验收标准需可验证、可量化、可复现，并覆盖正常/异常/边界场景
- 明确用户画像、关键流程、业务规则与约束
- 维护指标口径、风险依赖与 PRD 变更记录
- 功能列表需映射验收矩阵，保证可验证

## 输入

- 用户需求与约束

## 输出

- `PRD-{id}.md`（见 [requirements/](../../../requirements/)）

## 模板

- [templates/prd.md](../../../templates/prd.md)
