# 需求文档

## 命名规范

- 见 [conventions/](../conventions/)

## 模板

- [templates/prd.md](../templates/prd.md)
- PRD 需包含验收矩阵

## 验收标准定义（参考）

- 可验证：描述必须能通过测试或检查直接验证
- 可量化：包含明确条件/阈值/结果，不使用“尽量、尽快、良好”等模糊词
- 可复现：包含必要前置条件与输入
- 覆盖完整：至少包含正常流程、异常流程与边界条件

## 验收矩阵字段（建议）

- 功能点
- 前置条件
- 验收标准
- 验收方式（手动/自动/接口/集成）
- 用例ID（由测试阶段回填）
- 负责人
- 状态

## 验收矩阵填写规则（细化）

- 功能点：来自 PRD 功能列表，必须与功能列表一一对应
- 前置条件：环境、账号、数据、配置等可复现条件
- 验收标准：采用“前置条件/操作/预期结果（Given/When/Then）”描述
- 验收方式：
  - 手动：界面或业务流程验收
  - 自动：自动化用例覆盖
  - 接口：仅接口层验证
  - 集成：跨服务/跨模块联调
- 用例ID：测试阶段分配并回填，规则见 [conventions/](../conventions/)
- 负责人确定规则：
  - 默认：Tester（负责最终验收）
  - 需要业务确认：PM 参与或共同负责
  - 需要自动化：Developer 与 Tester 共同负责
- 状态：未开始 / 进行中 / 完成 / 阻塞

## PRD 结构补充（建议）

- 用户画像与场景
- 关键流程
- 业务规则与约束
- 版本规划（MVP/V1/V2）
- 指标与验收口径
- 风险与依赖
- 变更记录
- 术语表

## 功能与验收一致性

- 功能列表必须能映射到验收矩阵的至少一行
