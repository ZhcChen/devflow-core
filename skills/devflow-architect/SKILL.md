# 目标

- 产出技术方案并落盘至规范目录
- 明确架构、接口、数据模型与回滚策略

# 使用方式

1. 读取公共规范：[devflow-common](../devflow-common/SKILL.md) → [references/common.md](../devflow-common/references/common.md)
2. 读取角色职责：[docs/workflow/roles/architect/](../../docs/workflow/roles/architect/)
3. 使用模板：[docs/templates/spec.md](../../docs/templates/spec.md)
4. 技术方案落盘到：[docs/specs/](../../docs/specs/)
5. 方案确认后登记待开发清单（见 [docs/backlog/ready.md](../../docs/backlog/ready.md)）

# 输出

- `SPEC-{id}.md`
