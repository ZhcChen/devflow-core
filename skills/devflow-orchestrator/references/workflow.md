# 工作流程（精简版）

## 阶段流转

```
[需求分析] → [PRD确认] → [技术设计] → [方案确认] → [编码] → [审查] → [测试] → [完成]
```

## 角色分工

- Orchestrator（主 Agent）：流程调度、确认节点管理、状态维护
- PM（产品负责人）：需求拆解与 PRD
- Architect（架构师）：技术方案与架构设计
- Developer（开发）：按方案实现代码
- Reviewer（审查）：代码审查报告
- Tester（测试）：测试报告与缺陷清单

## 确认节点

- PRD确认：用户确认后进入技术设计
- 方案确认：用户确认后进入编码

## 回退原则

- 确认节点不通过则回退到上一阶段并携带修改意见
- 审查通过后进入测试，测试通过后进入完成
- 审查/测试不通过则回到编码阶段并携带问题清单
